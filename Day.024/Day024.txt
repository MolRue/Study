시험(월요일) 개발자 환경구축

1. 운영체제란 무엇인가? (개발자 환경구축.pdf 17p)

: 컴퓨터 시스템이 제공하는 하드웨어와 소프트웨어 기능을
 사용할수 있도록 해주는 시스템 소프트웨어이다.

2. 리눅스? 유닉스? 차이점 (역사는 필요없) (25p)

: 리눅스 오픈소스코드/ 무료/ 중대규모 서버에 이용
: 유닉스 오픈소스가 아님/ 라이선스 필요/ 대용량 처리, 안정성이 요구되는 서버에에 이용

3. CLI- 개발자 환경구축.pdf 25p
- command line interface
 : 사용자가 명령어를 입력하여 컴퓨터 시스템에 명령을 내리는 방식 
- graphic user interface
 : 마우스 등을 이용해 그레픽으로 표현된 윈도우 에서 명령을 실행

4. telnet - 개발자 환경구축.pdf 26p, ppt참고

 : 인터넷을 통하여 원격지의 호스트 컴퓨터에 접속할때 사용되는 인터넷 표준 프로토콜

5. 프로세스의 개념 (개발자 환경구축.pdf 33p, ppt참고)
 
 : 프로그램이 cpu를 할당받고 명령을 수행하고 있는 상태.


6. 병행 프로세스의 개념 (34p, ppt)

 : 겉으로는 동시에 실행되는거 같아 보이지만, 운영체재가 프로세서 시간을 나누어
  여러 프로세스를 동시에 실행하는 것처럼 보이게 하는 것이다.

7. 프로세스 상태 중 실행(run)이 어떤건지 (ppt)

 : 준비상태에 잇는 프로세스가 프로세서를 할당받아 실행되는 상태. 

8. 프로세스의 준비리스트 ( 개발자 환경구축.pdf 34p)

 : 우선순위를 부여하여 가장 높은 우선순위를 갖는 프로세스가 다음순서에 cpu를 할당받는것을 의미한다.

9. text(code)영역, data영역, heap영역, stack영역의 역할 - ppt참고

 code : 실행할 프로그램의 코드가 저장되는 영역
 data : 전역변수와 정적변수가 저장되는 영역, 프로그램이 종료되면 소멸
 heap : 사용자가 직접 관리할 수 있는 메모리 영역
 stack : 함수 호출과 관계되는 지역변수와 매게변수가 저장되는 영역. 함수 호출 완료후 소멸


이클립스 단축키
ctrl + -+ : 축소/확대
ctrl + F11 : 실행


이클립스 에서는 main 에 있는 코드가 가장 먼저 실행됨
